本手册针对 Windows 用户．

就我的体验而言，Sphinx_ 在构建文档的使用中，体现了极高的严谨水准（以及极客风格）；这一点是颇为难得的．相比于另一个文档构建软件 Gitbook_，Sphinx 基于 reStructureText 文件（可参考我的 `reST 笔记`_）而不是 Markdown．这固然使得学习成本上升，毕竟 Markdown 是更广为人知和使用的；但同时也消除了 Markdown 各家风格不同的弊病．

reST 格式不如 Markdown 易读（更不用提易写），但是允许用户自行定制风格；这点提供的便利在使用一段时间后就能有所体会．总之，各有所长吧．

在阅读本手册之前，建议：

* 已学习 reStructuredText 基本语法．前往：`reST 笔记`_．
